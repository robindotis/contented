<h2>Installation Guide</h2>
<h3>Getting started</h3>
<p>To use Contented a <a href="https://github.com/">GitHub</a> account is needed. Everything is done in GitHub: writing content, generating the site and publishing the pages. No other tool is needed other than a browser.</p>
<p>First of all sign up for a <a href="https://github.com/signup">GitHub account</a> and then login to GitHub.</p>
<p>Next make a copy of the DC code by forking the DC repository.</p>
<p>Notes:</p>
<ul>
<li>Start with simple setup of public rep names username.github.io and using branches for the deployment.</li>
<li>have advanced instructions on how to setup to run with private repos and external hosting.</li>
</ul>
<p>Now explore GitHub a little. The <a href="https://docs.github.com/en/get-started">GitHub docs</a> are very comprehensive. Being comfortable writing content and editing configuration files will be helpful, but this can also be learned. The <a href="https://docs.github.com/en/get-started/writing-on-github">GitHub Writing guide</a> is maybe a good place to start for those new to GitHub.</p>

<h3>File processing</h3>
<p>All content is stored in Markdown file, such as home.md.  Only files with the extension .md are processed by the engine. All other file types are ignored unless they are static simply files to be copied.</p>
<p>Each .md file is converted to an index.html file. This file is placed in a folder with the same path and name as the .md file, unless specified otherwise in the Front Matter configuration for the file using the &quot;permalink&quot; attribute.</p>
<p>For example the file:</p>
<pre><code>/posts/2023/summmer.md
</code></pre>
<p>would generated a file:</p>
<pre><code>/posts/2023/summmer/index.html
</code></pre>
<p>which would be accessible using the URL:</p>
<pre><code>/posts/2023/summmer/
</code></pre>
<p>And a file:</p>
<pre><code>/pages/home.md
</code></pre>
<p>With the following front matter attribute:</p>
<pre><code>permalink: /
</code></pre>
<p>Would generate file a here:</p>
<pre><code>/index.html
</code></pre>
<p>Which would be accessible using this URL:</p>
<pre><code>/
</code></pre>

<h3>File structure</h3>
<p>All Markdown files, whether posts or pages have two main parts.</p>
<p>Firstly, a Front Matter part, which contains the configuration details for the file and is always at the beginning of the file.</p>
<p>Secondly, the rest of the file which contains the content.</p>
<h4>Front Matter</h4>
<p>The Front Matter section of the file has a specific structure which <strong>must</strong> be followed.</p>
<p>Firstly it always starts and ends with the line: <code>---</code>.</p>
<p>Secondly it contains attribute / value pairs, one per line, eg: <code>title: Welcome to my blog</code>. The order of the attributes within the Front Matter is not important.</p>
<p>If the attribute contains a list of values, it would be spread over multiple lines, eg:</p>
<pre><code>tags:
- post
- travel
- photo 
</code></pre>
<p>The Front Matter of a typical post might therefore look something like this:</p>
<pre><code>---
title: Happy New Year
date: 2025-01-01 08:00:00
tags:
- celebration
- new year
- photo
---
</code></pre>
<p>DC understands the following Front Matter attributes. Any other attributes will be ignored.</p>
<table>
<thead>
<tr>
<th>Attribute</th>
<th>Value</th>
</tr>
</thead>
<tbody>
<tr>
<td>title</td>
<td>Free text</td>
</tr>
<tr>
<td>date</td>
<td>ISO format YYYY-MM-DD hh:mm:ss eg &quot;2025-01-01 08:00:00&quot;</td>
</tr>
<tr>
<td>permalink</td>
<td>Any valid URL path, eg &quot;/about/&quot;</td>
</tr>
<tr>
<td>tags</td>
<td>Free text list of tags, one per line</td>
</tr>
<tr>
<td>prefix</td>
<td>Free text to add before the page name in the title bar</td>
</tr>
<tr>
<td>others?</td>
<td>TBD</td>
</tr>
</tbody>
</table>
<p>Note: you do not need to enclose the value in quotes in the Front Matter section.</p>
<h3>Content</h3>
<p>In DC all content is written using <a href="https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax">GitHub's implementation of Markdown</a>.</p>
<p>Note that Markdown can also contain HTML. So any tags for which there is no Markdown equivalent can be written in HTML.</p>
<p>A typical Markdown content file might therefore look something like this:</p>
<pre><code>---
title: Happy New Year
date: 2025-01-01 08:00:00
tags:
- celebration
- new year
- photo

<h3>Archives</h3>
<p>Archives are lists of all the posts matching a specific tag.</p>
<p>They use the following Front Matter.</p>
<pre><code>---
title: Coding 
pagination:
  data: web
  alias: posts

---
</code></pre>
<p>data is tag name to include in the archive.</p>
<h3>Menus</h3>
<p>Menus can be defined for the header and footer. This is done in the file &quot;_data/menus.yaml&quot;. The syntax of this file is a little more complicated as it is written in YAML.</p>
<p><a href="https://www.yaml.info/learn/index.html">Learn more about the YAML syntax here</a>.</p>
<p>As an example this file menus.yaml file for a site with two menus. A header and a footer menu:</p>
<pre><code>- name: header
  items:
    - title: Home
      link: /home/
    - title: About
      link: /about/

<h3>Metadata</h3>
<p>Metadata for the site can be defined and is stored in the file &quot;_data/metadata.yaml&quot;.</p>
<p>This is an example the metadata.yaml file for a basic site:</p>
<pre><code>title: Robin's
url: https://www.robin.is/
urlShort: robin.is
language: en
description: Setup and host your own independent blog
startYear: 2025
relMe: https://github.com/robindotis/
gitHubRepo: https://github.com/robindotis/robin.is/
theme: sky
feed:
  subtitle: Setup and host your own independent blog
  filename: index.xml
  path: /feed/index.xml
  id: https://www.robin.is/
author:
  name: robindotis
  email: admin@robin.is
